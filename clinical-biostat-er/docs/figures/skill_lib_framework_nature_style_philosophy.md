# Design Philosophy: Clinical Cartography

This figure uses the visual language of a high-impact scientific systems diagram:
precise panels, restrained color, generous whitespace, and small typographic
signals that reward close inspection. The design should feel like an artifact
from a methods paper rather than a product slide: every line, junction, and label
has a purpose, and the complete page reads as a carefully mapped experimental
ecosystem.

Form is modular and directional. The primary workflow runs horizontally as a
clean procession of core skills; the evaluation loop wraps around it as a
separate evidence system. Rectangular modules represent governed instruction or
runtime surfaces, while rounded status capsules mark validation states. The
composition must make dependencies visible without becoming a flowchart maze.

Color is quiet and functional. Ink, slate, blue, green, amber, and red encode
architecture, executable workflow, validated evidence, pending frontier, and
unresolved semantic risk. The palette avoids decorative gradients and uses
scientific restraint: color is information, not ornament.

Scale and rhythm are journal-like. Panel letters, hairline rules, compact labels,
and numbered chips create a Nature-style explanatory figure. Text is sparse and
evidence-bearing: it names contracts, artifacts, and status boundaries without
turning the figure into prose.

The final piece should look meticulously crafted, with master-level alignment and
spacing. It should communicate that this skill library is not merely a folder
tree, but a governed agent-execution system with runtime code, reproducibility
tests, ClaudeCode behavior cases, and a known current frontier.
